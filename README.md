# YouTube Downloader GUI

YouTube 단일 영상 또는 재생목록을 GUI로 다운로드하는 PySide6 기반 데스크톱 도구입니다.

이 앱은 독립 실행형 프로젝트입니다. 기존 참고 코드 폴더인 `YT-Playlist-Downloader` 또는 `yt_av.py`, `yt_avsl.py`가 없어도 이 저장소만으로 실행됩니다.

## 주요 기능

- 단일 영상 URL 다운로드
- 재생목록 URL 다운로드
- 영상 MP4 다운로드
- 음원 MP3 추출
- 영상 화질 선택: 최고화질, 2160p, 1440p, 1080p, 720p, 480p, 360p 이하
- 저장 폴더 선택 및 마지막 선택 기억
- 다운로드 전 재생목록 미리보기
- 재생목록 미리보기에서 제목, 길이, 예상 파일명 표시
- 여러 URL 대기열 추가
- 대기열 순서 변경 및 항목 삭제
- 재생목록명 하위 폴더 자동 생성 및 폴더명 직접 수정
- 파일명 형식 선택: 제목만, `001_` 번호 붙이기
- 중복 파일 처리 선택: 건너뛰기, 덮어쓰기, 새 이름으로 저장, 다운로드 기록 기준 건너뛰기
- 설정 변경 시 예상 파일명 실시간 미리보기
- 다운로드 후 예상 파일과 실제 파일 자동 검증
- 누락, 0바이트, 파일명 차이 항목을 결과 검증 탭에서 확인
- 다운로드 세션 로그 자동 저장
- 창 크기, 좌우 분할, 탭 위치, 테이블 컬럼 순서와 너비 저장
- 현재 항목 진행률, 전체 진행률, 로그, 취소 버튼
- 다운로드 완료 알림
- Python, yt-dlp, ffmpeg 상태 표시

재생목록 URL 또는 `list=`가 포함된 YouTube URL을 넣으면 전체 재생목록 다운로드를 우선합니다. 화질은 선택한 해상도 이하에서 각 영상이 제공하는 가장 적절한 포맷을 `yt-dlp`가 자동으로 선택합니다.

## 실행 방법: Windows

1. `run_app.bat`를 더블클릭합니다.
2. 처음 실행할 때 `.venv` 가상환경을 만들고 `PySide6`, `yt-dlp`를 설치합니다.
3. 앱이 열리면 YouTube URL, 저장 위치, 다운로드 형식을 선택합니다.
4. 영상으로 받을 경우 원하는 화질을 선택합니다.
5. `목록 불러오기`를 눌러 받을 영상을 확인합니다.
6. 재생목록 폴더 생성, 폴더명, 파일명 형식, 중복 처리 방식을 확인합니다.
7. `대기열 추가`를 눌러 다운로드 목록에 넣습니다.
8. 필요한 URL을 더 추가한 뒤 대기열 순서를 조정하거나 항목을 삭제합니다.
9. `대기열 다운로드 시작`을 누릅니다.
10. 다운로드가 끝나면 `결과 검증` 탭에서 정상, 누락, 크기 이상, 파일명 차이 항목을 확인합니다.

## 실행 방법: macOS / Linux

터미널에서 프로젝트 폴더로 이동한 뒤 아래 명령을 실행합니다.

```bash
chmod +x run_app.sh
./run_app.sh
```

처음 실행할 때 `.venv` 가상환경을 만들고 필요한 Python 패키지를 설치합니다.

## 필요한 도구

- Python 3.10 이상
- `yt-dlp`: `run_app.bat`가 자동 설치합니다.
- `ffmpeg`: MP3 변환 또는 고화질 영상 병합에 필요합니다.
- Node.js, Deno 또는 Bun: 최신 YouTube 추출에서 JavaScript 런타임이 필요할 수 있습니다.

## ffmpeg 설치

MP3 변환과 고화질 MP4 병합에는 `ffmpeg`가 필요합니다. Windows에서는 ffmpeg를 설치한 뒤 `bin` 폴더를 Windows `PATH`에 추가하세요.

macOS에서는 Homebrew를 사용하는 경우 아래 명령으로 설치할 수 있습니다.

```bash
brew install ffmpeg
```

Linux에서는 배포판 패키지 관리자를 사용하세요.

```bash
sudo apt install ffmpeg
```

앱은 실행할 때 `yt-dlp`를 최신 버전으로 업데이트합니다. 또한 Node.js, Deno, Bun 중 설치된 JavaScript 런타임을 감지해 `yt-dlp --js-runtimes ... --remote-components ejs:github` 옵션을 자동으로 사용합니다.

## 다운로드 실패 확인

`yt-dlp가 오류 코드 1로 종료되었습니다.`는 yt-dlp가 다운로드를 실패했다는 공통 종료 코드입니다. 앱은 실패 시 원인으로 보이는 `ERROR`, `WARNING`, 최근 로그를 팝업과 로그 창에 함께 표시합니다.

다운로드가 대부분 정상으로 보이더라도 재생목록 안의 일부 비공개, 삭제, 제한 영상 때문에 오류 코드가 반환될 수 있습니다. 이 경우 `결과 검증` 탭에서 실제로 누락된 파일이 있는지 확인하세요. 앱은 다운로드 시점의 예상 파일명과 저장 폴더를 기준으로 파일 존재 여부와 0바이트 파일 여부를 검사합니다.

세션 로그는 선택한 저장 위치의 `_download_reports` 폴더에 저장됩니다.

자주 발생하는 원인:

- 재생목록 안에 비공개, 삭제, 지역 제한, 연령 제한 영상이 포함된 경우
- YouTube 재생목록 URL을 복사하는 과정에서 `list=` 값 끝 문자가 빠진 경우
- 선택한 화질이나 포맷을 해당 영상이 제공하지 않는 경우
- YouTube 추출 방식이 바뀌어 yt-dlp 업데이트가 필요한 경우
- ffmpeg 병합 또는 MP3 변환 중 오류가 난 경우
- 저장 폴더 권한, 긴 경로, 디스크 용량 문제가 있는 경우

앱은 `list=` 값 끝의 `_`가 누락되어 YouTube가 `HTTP Error 400: Bad Request`를 반환하는 흔한 경우를 자동 보정해 다시 분석합니다. 보정에 성공하면 URL 입력칸도 정상 URL로 바뀝니다.

## 파일명과 중복 처리

파일명 형식:

- 폴더 `재생목록명 폴더 자동 생성` + 파일명 `제목만`: `재생목록명/영상제목.mp4`
- 폴더 `재생목록명 폴더 자동 생성` + 파일명 `001_번호 + 제목`: `재생목록명/001_영상제목.mp4`
- 폴더 `선택 폴더에 바로 저장` + 파일명 `001_번호 + 제목`: `001_영상제목.mp4`

`목록 불러오기`를 누르면 폴더명 입력칸에 재생목록명이 자동으로 들어갑니다. 이미 같은 이름의 폴더를 사용자가 직접 선택해 둔 경우에는 폴더 옵션을 `선택 폴더에 바로 저장`으로 바꾸면 중첩 폴더가 생기지 않습니다. 폴더명을 직접 수정하면 받을 목록 미리보기의 예상 파일명과 실제 저장 경로에 같이 반영됩니다.

중복 처리:

- `이미 있으면 건너뛰기`: 같은 파일명이 있으면 덮어쓰지 않습니다.
- `덮어쓰기`: 기존 파일을 교체합니다.
- `새 이름으로 저장`: 파일명 뒤에 실행 시각 토큰을 붙여 새 파일로 저장합니다.
- `다운로드 기록 기준 건너뛰기`: 저장 폴더의 `.download_archive.txt` 기록을 기준으로 이미 받은 영상을 건너뜁니다.

## 설정 저장 위치

앱은 마지막 저장 폴더와 선택한 형식을 아래 파일에 저장합니다.

```text
%APPDATA%\YoutubeDownloaderGui\settings.json
```
